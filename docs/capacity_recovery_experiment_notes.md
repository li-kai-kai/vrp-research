# 道路容量渐进恢复实验记录

实验日期：2026-06-09

本文记录第一版实验原型，用于验证两个拟定创新点是否可以落到可运行程序中：

1. 道路状态不只是二元可达，而是容量随修复进度逐步恢复。
2. 使用 NSGA-II + ALNS 混合算法求解修复-配送协同优化问题。

## 1. 实验入口

新增脚本：

```bash
scripts/reproduce/capacity_recovery.py
```

快速运行：

```bash
uv run python scripts/reproduce/capacity_recovery.py \
  --scenario both \
  --seeds 1 \
  --sim-nodes 25 \
  --pop-size 24 \
  --generations 20 \
  --alns-iterations 8 \
  --output-dir outputs/capacity_recovery
```

输出文件：

| 文件 | 内容 |
|---|---|
| `outputs/capacity_recovery/runs.csv` | 每个场景的代表解汇总 |
| `outputs/capacity_recovery/solutions.json` | 修复顺序、队伍分配、目标值和指标 |
| `outputs/capacity_recovery/convergence.csv` | 每代收敛记录 |

## 2. 当前实现范围

本轮实验是原型验证，不是最终论文级求解器。

已经实现：

- 随机仿真路网；
- 汶川 38 节点路网；
- 16 条汶川受损路段；
- 5 阶段道路容量恢复；
- 4 类救援车辆通行阈值；
- 多周期修复进度更新；
- 车型相关动态最短路；
- NSGA-II 非支配排序与拥挤距离；
- ALNS 局部改进算子。

暂未实现：

- 完整 BPR 拥堵函数；
- 普通交通 OD 流；
- 维修队真实移动路径；
- 完整 Pareto 前沿导出；
- 多种算法系统对比；
- 多随机种子统计检验。

## 3. 模型原型说明

### 3.1 道路修复进度

每条受损路段的修复进度为：

```text
p_a^t = 已投入维修时间 / 完全修复时间
```

若路段在周期结束时处于 `0 < p_a^t < 1`，则记录为部分恢复状态。

### 3.2 容量恢复阶段

| 修复进度 | 状态 | 容量比例 |
|---|---|---:|
| `[0, 0.30)` | 完全中断 | 0.00 |
| `[0.30, 0.60)` | 临时抢通 | 0.30 |
| `[0.60, 0.80)` | 单车道限行 | 0.60 |
| `[0.80, 1.00)` | 基本恢复 | 0.80 |
| `1.00` | 完全恢复 | 1.00 |

### 3.3 车辆通行阈值

| 车辆类型 | 载重 | 数量 | 最低通行进度 |
|---:|---:|---:|---:|
| 1 | 5 吨 | 150 | 0.30 |
| 2 | 10 吨 | 110 | 0.50 |
| 3 | 15 吨 | 70 | 0.70 |
| 4 | 20 吨 | 40 | 0.80 |

该设置用于表达“小车先通，大车后通”。

## 4. 本轮结果

运行命令：

```bash
uv run python scripts/reproduce/capacity_recovery.py --scenario both --seeds 1 --sim-nodes 25 --pop-size 24 --generations 20 --alns-iterations 8 --output-dir outputs/capacity_recovery
```

结果摘要：

| 场景 | 节点数 | 受损边 | 总满足率 | 最小满足率 | 最终修复比例 | 部分恢复路段-周期数 | 小型车配送占比 | 运行时间 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 随机仿真 | 25 | 19 | 0.900 | 0.900 | 0.158 | 18 | 0.000 | 6.05s |
| 汶川路网 | 38 | 16 | 0.797 | 0.797 | 1.000 | 13 | 0.411 | 7.92s |

解释：

- 随机仿真中最终总满足率达到供给约束附近，且最小满足率同步达到 0.900，说明公平配送约束没有被局部贪心破坏。
- 汶川路网中最终修复比例为 1.000，说明 72 小时内当前简化排班可完成全部受损路段修复。
- 汶川路网的总需求为 11288、总供应为 9000，因此公平满足率上限为 `9000 / 11288 = 0.7973`。表中的 0.797 不是算法额外提升出来的结果，而是供应不足条件下的理论上限。
- 汶川路网的小型车配送占比为 0.411，说明车型通行阈值已经影响配送选择。
- 两个场景都有部分恢复路段-周期记录，说明容量渐进恢复逻辑已经实际参与评估，而不是只在文档中存在。

## 5. 初步观察

1. 容量渐进恢复机制可以跑通。实验中出现了部分恢复路段，且这些路段会影响不同车型的可行路径。
2. 修复后的配送评估采用公平优先分配，避免早期可达节点过度占用供应，汶川路网最低满足率不再出现 0。
3. 当前 ALNS 算子较轻，主要用于修复顺序、队伍分配和配送优先级扰动，还没有专门针对未满足面积和晚期开通节点设计强修复算子。
4. 当前输出只是代表解，不等于完整 Pareto 分析。后续应导出 Pareto front 并选择不同偏好方案。

## 6. 下一步实验建议

优先级从高到低：

1. 导出完整 Pareto 前沿，而不是只输出一个代表解。
2. 增加公平性导向 ALNS 算子，例如优先插入最低满足率需求点。
3. 增加对照组：
   - 二元道路状态 + NSGA-II；
   - 容量渐进恢复 + NSGA-II；
   - 容量渐进恢复 + NSGA-II + ALNS。
4. 对汶川路网运行多随机种子，报告均值和标准差。
5. 将通行时间从 `free_time / capacity_ratio` 升级为 BPR 函数。
6. 增加图形输出：收敛曲线、满足率曲线、道路恢复阶段热力图。

## 7. 当前结论

第一版实验说明：两个创新点具备可实现性。容量渐进恢复可以自然连接修复进度、道路容量、车型可通行性和配送路径；NSGA-II + ALNS 可以作为多目标搜索框架继续扩展。

但当前结果只能作为原型验证，不能直接作为论文实验结果。正式实验需要加入对照组、多种子统计、Pareto 前沿分析和更完整的局部搜索算子。
