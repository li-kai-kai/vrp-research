## Whole-Thesis Law

Treat each chapter as an input-output unit:

```text
previous chapter output -> current chapter task -> method/action -> result -> next chapter input
```

The recurring hidden chain is:

```text
gap -> measurable variables/indicators -> model/path -> comparison/dynamic analysis -> management suggestion
```

Do not write chapters as independent essays. Each chapter should explicitly inherit the previous chapter and prepare the next one.

## Abstract Law

Write the abstract as a compressed version of the full thesis:

1. Practical background and problem.
2. Existing research gap.
3. Research object, data, and method chain.
4. Main work in `首先 / 其次 / 最后` order.
5. Main results and application value.

Common move:

```text
现实问题 -> 既有研究不足 -> 本文以[对象]为基础 -> 采用[方法链] -> 结果表明[核心发现] -> 可为[管理/政策/设计]提供依据
```

## Chapter 1 Laws

### Research Background

Write from large to small:

```text
macro policy/industry trend -> field problem -> specific object -> practical contradiction -> need for research
```

Avoid starting directly with the model. The background should make the topic necessary before methods appear.

### Research Purpose

Purpose is not a slogan. It should include object, method chain, and target outcome:

```text
为明确[机制/关系/影响程度]，本文以[对象]为研究对象，采用[方法1]-[方法2]-[方法3]，实现对[过程/机制/效果]的定量描述，并为[管理目标]提供依据。
```

### Research Significance

Write two levels:

- Theoretical significance: fill a factor/object/method gap, enrich a model application, extend a framework.
- Practical significance: provide policy, management, design, safety, resource allocation, or decision support.

Good pattern:

```text
在理论层面，本文通过[新变量/新对象/新模型组合]，拓展了[理论/模型]在[场景]中的应用。
在实践层面，本文基于[数据/结果]，为[主体]制定[政策/管理措施]提供参考。
```

### Research Content

Research content usually has three numbered tasks that mirror Chapters 3-5:

1. Build variables, factors, indicators, questionnaire, or scene.
2. Build or validate mechanism/model.
3. Compare, simulate, analyze effects, predict, or propose strategy.

Each item should include `what is built + how it is built + what it supports`.

### Research Methods

Do not list methods generically. Explain each method's role in the chain:

```text
[方法]用于[数据/变量/模型动作]，为后续[下一步]提供[数据/参数/假设/依据]。
```

### Technical Route

Technical route is a process diagram in prose. Use action verbs:

```text
文献梳理 -> 确定因素/指标 -> 数据采集 -> 质量检验 -> 模型构建 -> 结果分析 -> 策略建议
```

In text before the figure, summarize the route in one paragraph. Do not explain every box unless needed.

### Innovation Points

Innovation points are written as `existing limitation -> thesis improvement -> expected contribution`.

Common innovation types:

- New factor: introduce a neglected latent, dynamic, behavioral, or infrastructure variable.
- New object/context: study a specific group, region, building, industry, or scenario.
- New method chain: combine models so one model's output becomes another model's input.
- New application: apply an existing model to a less-studied management problem.

Avoid claiming "first" unless the evidence is strong.

## Chapter 2 Law

Chapter 2 is not another literature review. It defines the conceptual and theoretical tools used later.

Use this sequence:

```text
concept definition -> this thesis boundary -> supporting theory -> model principle -> support for later chapters
```

For each concept:

1. Give a general definition.
2. State what it means in this thesis.
3. Explain which variable, indicator, hypothesis, or model it supports.

For each theory/model:

1. Explain the basic idea.
2. State why it fits the research problem.
3. Point to the later chapter where it will be used.

End Chapter 2 by saying it "奠定理论基础" only after naming the exact later use.

## Chapter 3 Law

Chapter 3 turns the research question into measurable material.

Choose the relevant path:

- Questionnaire path: variable source -> dimension/item design -> pre-survey -> formal survey -> sample description -> reliability/validity/factor analysis.
- Indicator path: selection principle -> literature/expert source -> Delphi/scoring -> screening criteria -> final indicator system -> weight or consistency preparation.
- Factor-intervention path: theory/literature factor extraction -> expert scoring -> DEMATEL-ISM or relation analysis -> path basis.
- Simulation scene path: object selection -> scene attributes -> parameter source -> evaluation indicators.

Hidden law:

```text
make variables credible before using them in models
```

End by stating that the data/indicator/factor system is reliable enough for Chapter 4.

## Chapter 4 Law

Chapter 4 is the mechanism/model chapter.

Use this sequence:

```text
previous data/factors -> hypothesis/model framework -> variable definition/equations -> estimation/test -> result interpretation -> mechanism conclusion
```

For SEM/HCM/Logit-type chapters:

- Explain why the model is chosen.
- Define variables and hypotheses.
- Report fit or diagnostic indicators.
- Interpret direction, strength, and meaning.
- If comparing models, explain what the improved model captures that the baseline misses.

For each result, use:

```text
statistical result -> hypothesis/model implication -> real-world explanation -> next-step use
```

Do not write a table dump. The table supports the mechanism, not the other way around.

## Chapter 5 Law

Chapter 5 deepens Chapter 4 and turns results toward practice.

Common roles:

- Model comparison: baseline vs improved model.
- Dynamic simulation: static path coefficients become dynamic parameters.
- Effect analysis: identify key variables, lagged effects, or secondary indicators.
- Management strategy: convert key findings into measures.

Opening pattern:

```text
基于第[数字]章构建的[模型/指标/参数]，本章进一步开展[实证/仿真/影响因素]分析，以识别[关键机制/作用效果]。
```

Recommendation law:

```text
finding -> affected object -> concrete measure -> expected management effect
```

Do not introduce recommendations before the evidence has been analyzed.

## Chapter 6 Law

Chapter 6 echoes the whole thesis.

Opening:

```text
本文以[对象]为研究对象，围绕[核心问题]，通过[方法链]开展研究，主要得出以下结论。
```

Conclusion order should match research content order:

1. Data/questionnaire/indicator/factor construction conclusion.
2. Mechanism/model/path conclusion.
3. Comparison/simulation/effect conclusion.
4. Optional key management implication.

Suggestions should be grouped by key findings, not by random policy slogans.

Outlook should be honest and bounded:

- sample or region limitation.
- variable or indicator limitation.
- model assumption or method limitation.
- data period or dynamic validation limitation.

Future work should directly respond to those limitations.

## Chapter Summary Law

Use the chapter summary to transfer value to the next chapter.

Structure:

```text
本章首先[做了什么]；其次[做了什么]；最后[做了什么]。结果表明[核心发现]，为后续[下一章任务]提供了[数据/参数/假设/理论]基础。
```

For model chapters, include the main mechanism result. For data chapters, include reliability/validity or representativeness. For simulation/effect chapters, include comparison or key factor results.
